WHAT WAS CREATED:
==================

1. daily_inventory_sync.py

   - Updates all color variants with current quantities
   - Logs all activity

2. setup_daily_sync.bat
   - Windows batch file to run the sync
   - Activates virtual environment automatically
   - Logs output to logs/daily_sync.log


HOW TO SET UP DAILY AUTO-SYNC:
===============================

OPTION 1: Windows Task Scheduler (Recommended)
-----------------------------------------------

1. Open Task Scheduler:
   - Press Win + R
   - Type: taskschd.msc
   - Press Enter

2. Create New Task:
   - Click "Create Basic Task" in right sidebar
   - Name: "Purposefully Made KC Inventory Sync"
   - Description: "Daily sync of product inventory from S&S Activewear"
   - Click Next

3. Set Trigger:
   - Select "Daily"
   - Click Next
   - Set time: 2:00 AM (or when your store is closed)
   - Click Next

4. Set Action:
   - Select "Start a program"
   - Click Next
   - Program/script: Browse and select setup_daily_sync.bat
   - Start in: C:\Users\Kayle\OneDrive\Desktop\kb_apparel_site
   - Click Next

5. Finish:
   - Review settings
   - Check "Open the Properties dialog"
   - Click Finish

6. In Properties:
   - Security options tab
   - Select "Run whether user is logged on or not"
   - Check "Run with highest privileges"
   - Click OK


OPTION 2: Manual Daily Sync (Simple)
-------------------------------------

Just run this command once a day:

   cd C:\Users\Kayle\OneDrive\Desktop\kb_apparel_site
   .\venv\Scripts\python daily_inventory_sync.py

Or double-click: setup_daily_sync.bat


OPTION 3: Admin Panel Sync (Already Built!)
--------------------------------------------

You can also sync manually anytime via:

2. Go to Products
3. Click "Sync from S&S Activewear"
4. Click "Start Sync"


WHAT THE DAILY SYNC DOES:
==========================

Every day at 2:00 AM (or your chosen time):

1. Connects to S&S Activewear API
2. Fetches current inventory for all products
3. Updates quantity for each size in each color
4. Saves to database
5. Logs results to logs/daily_sync.log

This keeps your store's inventory ALWAYS accurate!
